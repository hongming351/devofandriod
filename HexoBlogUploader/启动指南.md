## 3. 构建和运行应用

### 步骤 1：同步 Gradle

1. 打开项目后，Android Studio 会自动开始同步 Gradle
2. 如果同步失败，点击 "Sync Now" 或 "Try Again"
3. 等待所有依赖下载完成

### 步骤 2：连接设备或模拟器

#### 选项 A：使用物理设备

1. 启用手机的开发者选项：
   - 进入设置 → 关于手机 → 连续点击 "版本号" 7次
   - 返回设置 → 开发者选项 → 启用 USB 调试
2. 通过 USB 连接手机到电脑
3. 在手机上允许 USB 调试

#### 选项 B：使用 Android 模拟器

1. 在 Android Studio 中打开 AVD Manager：
   - 点击工具栏的 AVD Manager 图标
   - 或通过 Tools → AVD Manager
2. 创建新的虚拟设备：
   - 点击 "Create Virtual Device"
   - 选择设备型号（推荐 Pixel 5）
   - 选择系统镜像（推荐 Android 11+）
   - 完成创建
3. 启动虚拟设备

### 步骤 3：运行应用

1. 确保设备已连接（物理设备或模拟器）
2. 点击工具栏的 "Run" 按钮（绿色三角形）
3. 或使用快捷键：Shift + F10
4. 选择目标设备
5. 等待应用构建和安装

## 4. 首次使用配置

### 1. 克隆你的 Hexo 博客仓库

1. 打开应用后，点击右下角菜单按钮
2. 选择 "Git 管理"
3. 点击 "克隆我的博客"
4. 输入你的 GitHub 信息：
   - 仓库URL：`https://github.com/你的用户名/你的用户名.github.io.git`
   - GitHub Token：从 <https://github.com/settings/tokens> 获取

### 2. 配置 GitHub Actions（可选）

1. 在 Git 管理界面点击 "设置 GitHub Actions"
2. 系统会自动生成配置文件
3. 提交并推送配置文件到 GitHub

### 3. 开始写作

1. 返回主界面
2. 点击右下角 "+" 按钮创建新文章
3. 使用 Markdown 编辑器编写内容
4. 保存文章并自动提交到 Git

## 5. 常见问题解决

### 问题 1：Gradle 同步失败

**解决方案：**

```bash
# 清理 Gradle 缓存
cd d:\devofandriod\HexoBlogUploader
gradlew clean

# 重新同步
gradlew build
```

### 问题 2：无法连接到设备

**解决方案：**

1. 检查 USB 调试是否启用
2. 重新插拔 USB 线
3. 重启 ADB 服务：

```bash
adb kill-server
adb start-server
```

### 问题 3：应用崩溃

**解决方案：**

1. 查看 Logcat 错误信息
2. 检查 AndroidManifest.xml 权限
3. 确保所有依赖已正确下载

### 问题 4：Git 操作失败

**解决方案：**

1. 检查网络连接
2. 验证 GitHub Token 是否正确
3. 确保仓库地址正确
